3. PIPES
---------------------------------------------------------------

STATUS: Partially implemented. Custom pipe created. TitleCase pipe added.

ALREADY PRESENT:
  - number pipe   : home.ts uses {{ radius / 1000 | number:'1.0-1' }}, {{ r.averageRating | number:'1.1-1' }}

WHAT WAS MISSING:
  The formatReviewDate() method in restroom-detail.ts manually formatted
  ISO date strings into user-friendly relative labels (e.g., "2 hours ago").
  This was 35+ lines of inline component logic that could not be reused.

CHANGE MADE — Custom Pipe Created:
  File: frontend/src/app/pipes/recent-date.pipe.ts
  Pipe: RecentDatePipe  (selector: 'recentDate')

  Behavior:
    - Transforms an ISO date string to a human-friendly relative + absolute label
    - Example output: "2 hours ago • Mar 23, 2026, 9:30 PM"
    - Returns "Date unavailable" for null/invalid input

  Applied in:
    File: restroom-detail.ts
    Before: {{ formatReviewDate(r.createdAt) }}
    After:  {{ r.createdAt | recentDate }}

  The formatReviewDate() method was removed from the component entirely,
  keeping the component clean and the logic reusable as a proper Angular pipe.

CHANGE MADE — Built-in TitleCase Pipe Applied:
  File: restroom-detail.ts
  Before: {{ a }}
  After:  {{ a | titlecase }}

  Applied to amenity tags in the restroom detail page so amenity names
  (e.g., "child friendly") are always displayed in Title Case regardless
  of how they are stored in the database.

4. SERVICES
---------------------------------------------------------------

STATUS: Already uses services for API and auth. Shared logic centralized.

ALREADY PRESENT:
  - ApiService      : All HTTP API calls centralized (api.service.ts)
  - AuthService     : Login state, token, user role management
  - LocationService : Geolocation tracking, HAU default coordinates
  - DatetimeService : PH timezone clock signal

WHAT WAS MISSING:
  The following methods were DUPLICATED in both home.ts and restroom-detail.ts:
    1. isRestroomOpenNow(r: Restroom): boolean
       - 30+ lines of PH timezone open/close logic
    2. getDistance(lat1, lon1, lat2, lon2): number
       - Haversine formula for distance in meters

  Having two copies of the same logic violates DRY and makes maintenance risky.

CHANGE MADE — Moved shared methods to LocationService:
  File: frontend/src/app/services/location.service.ts

  Methods added:
    - isRestroomOpenNow(r: Restroom): boolean
        Uses Asia/Manila timezone to check if a restroom is currently open.
    - getDistance(lat1, lon1, lat2, lon2): number
        Returns distance in meters using the Haversine formula.

  Both components now delegate to the service:
    - home.ts:            this.locationService.isRestroomOpenNow(r)
    - restroom-detail.ts: this.locationService.isRestroomOpenNow(r)
    - home.ts:            this.locationService.getDistance(...)
    - home.ts (filter):   this.locationService.getDistance(...)

  Local duplicate methods removed from home.ts and restroom-detail.ts.
  Components expose an arrow-function wrapper for template binding:
    isRestroomOpenNow = (r: Restroom) => this.locationService.isRestroomOpenNow(r);
